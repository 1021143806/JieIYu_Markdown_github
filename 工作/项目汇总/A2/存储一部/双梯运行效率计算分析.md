# 新增一个电梯，双梯方案对比

## 效率对比

### **效率对比：双梯专用 vs 双梯通用**

#### **1. 双梯专用方案（一上一下）**

- **上行电梯**：
`102s = 51s（上行任务） + 51s（空载回1楼）` → 完成1次上行任务

- **下行电梯**：
`102s = 51s（下行任务） + 51s（空载回2楼）` → 完成1次下行任务

- **并行效果**：
每102s完成 **1次上行 + 1次下行**（共2次任务）

- **总耗时（310上+310下）**：
`310 × 102s = 31,620s`（8.78小时）

#### **2. 双梯通用方案（可上可下）**

- **连续执行反向任务**：


任务1（上行）：1楼→2楼（51s）
任务2（下行）：2楼→1楼（51s）→ 省去复位时间！



- **完整上下行周期**：`51s + 51s = 102s` → 完成 **2次任务**（一上一下）

- **并行效果**：
双梯每102s可完成 **2次上行 + 2次下行**（共4次任务）

- **总耗时（310上+310下）**：
`310 × 102s / 2 = 15,810s`（4.39小时）

> ✅ **关键发现**：
通用方案通过**消除空载复位**，在相同102s周期内，任务吞吐量是专用方案的 **2倍**！

### **结论与建议**

1. **通用方案绝对优势**：

  - 耗时仅4.39h（vs 专用方案8.78h），**效率提升100%**。

  - 通过**任务反向衔接**，将专用方案中浪费的复位时间转化为有效运力。

2. **专用方案致命缺陷**：

  - 50%时间浪费在空载复位上（51s/102s），导致效率减半。

3. **实施关键**：

  - **动态任务分配**：确保电梯抵达楼层时，AGV已就绪（如2楼备好下行AGV）。

  - **流量监控**：若某方向任务不足，调度系统可临时切换电梯方向（如上行梯支援下行）。

4. **效益预测**：

  - 当前日任务620次仅需4.39h，剩余5.61h可支持 **+425次任务**（总1045次）。

  - 若未来任务量增长（如1200次/日），通用方案仍可轻松应对（需7.02h）。

## 路线对比

两梯一上一下与两梯均可上下，效率均可满足一天所有电梯任务的需求。

### 方案一：两梯一梯仅上，一梯仅下

- 一上一下方案可将备料车路线与滚筒路线完全隔离，实现线性路程循环。

- 路程较长

- 无交通管制时间，路线隔离

- 对于一次性全是上梯或一次性全是下梯任务的效率提升较小，与单电梯效率几乎无差别


### 方案二：两梯均可上下

- 路线复杂

- 交通管制时间较长，会影响滚筒路线运行

- 电梯相关任务效率最大化，任务承载能力翻倍

- 对于高峰时期任务可以更快处理



推荐选择方案一，目前瓶颈主要为三楼滚筒举升路线混用导致的交通管制压力，使用方案一可以完全隔离路线，同时电梯任务执行数量可满足一天内要求。
