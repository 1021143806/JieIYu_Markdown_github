## 📝 命名规范

### 目录命名
- 格式：`数字前缀-中文名称`
- 数字前缀用于排序，建议使用两位数字
- 名称简洁明了，便于理解

### 文件命名
- 避免使用特殊字符：`< > : " | ? *`
- 建议使用有意义的中文或英文名称
- 日期格式建议使用：`YYYY-MM-DD`
- 版本控制建议使用：`v1.0`, `v2.1` 等

### 资源文件命名
- 图片文件：保持原始名称，必要时添加前缀
- 文档模板：使用统一的模板前缀
- 脚本文件：使用描述性名称，包含功能说明

## 🔧 使用工具

### 重构脚本
- **文件名**: `目录重构脚本.py`
- **功能**: 将现有目录结构重构为新的知识库结构
- **使用方法**: 运行脚本并确认操作

### 验证脚本
- **文件名**: `结构验证脚本.py`
- **功能**: 验证目录结构的完整性和规范性
- **使用方法**: 定期运行以检查结构健康度

## 📚 管理建议

### 日常维护
1. **定期清理**: 每月清理一次 07-临时文件 目录
2. **分类整理**: 新增文件时及时分类到相应目录
3. **命名规范**: 严格按照命名规范创建文件和目录
4. **备份重要**: 定期备份重要的工作文档和学习笔记

### 扩展原则
1. **保持结构**: 新增内容时尽量使用现有目录结构
2. **合理分类**: 如需新增目录，确保分类合理且有明确用途
3. **层级控制**: 保持目录层级在3-4层以内
4. **文档同步**: 为重要目录创建 README.md 说明文件

### 质量控制
1. **定期验证**: 使用验证脚本检查结构完整性
2. **内容审核**: 定期检查内容的准确性和时效性
3. **重复清理**: 及时清理重复和过时的内容
4. **链接维护**: 检查和更新文档中的内部链接

## 🚀 快速开始

### 第一次使用
1. 运行 `目录重构脚本.py` 重构现有目录
2. 运行 `结构验证脚本.py` 验证重构结果
3. 查看生成的报告了解迁移详情
4. 根据建议进行进一步优化

### 日常使用
1. 新增文件时按照分类规则存放
2. 定期运行验证脚本检查结构
3. 及时整理临时文件目录
4. 保持命名规范的一致性

## 📊 监控指标

### 结构健康度
- 目录完整性：所有预期目录都存在
- 命名规范性：文件和目录命名符合规范
- 深度合理性：目录层级控制在合理范围
- 分布均衡性：文件在各目录间分布合理

### 使用效率
- 查找速度：能够快速定位所需文件
- 分类准确性：文件分类正确无误
- 维护成本：日常维护工作量适中
- 扩展能力：结构便于后续扩展

## 🔍 故障排除

### 常见问题
1. **目录缺失**: 运行重构脚本补充缺失目录
2. **命名不规范**: 手动重命名或使用批量重命名工具
3. **文件错位**: 手动移动文件到正确目录
4. **结构过深**: 考虑简化目录层级

### 恢复方案
1. **备份恢复**: 使用备份恢复误删内容
2. **脚本修复**: 运行修复脚本调整结构
3. **手动整理**: 对于复杂问题进行手动整理
4. **专业工具**: 使用专业的文件管理工具辅助
