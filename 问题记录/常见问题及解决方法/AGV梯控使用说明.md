# RCS 及 AGV 电梯使用说明

入口：设备管理 - 执行设备管理 - 电梯/提升机

### 1.1 可进行以下操作

- 电梯状态查看

  - 设备编号
  - IP 地址
  - 是否在线

- 电梯是否启用

  - 可对电梯进行禁用，被禁用后 AGV 将不会使用电梯

- 电梯释放

  - 适用于设备在电梯处异常离线，电梯显示有任务导致无法使用的情况，可点击电梯释放进行任务解除。

- 电梯参数编辑

  ![image-20240327164456095](D:\VScode\gitcode\JieIYu_Markdown_1\问题记录\常见问题及解决方法\AGV梯控使用说明.assets\image-20240327164456095.png)

- 主要配置项

  - 设备 IP
    - 按实际及现场需求配置
  - 设备名称
    - 可自定义
  - 设备端口
    - 按实际及现场需求配置
  - 电梯交接区
    - 电梯点位，要求码值为 00 开头的八位数字码，不推荐换码，建议多打印以备用更换。
  - 是否启用
  - 提前申请召梯
    - 可设置在AGV到达等待点前提前下发呼梯，合理设置可提高效率

![image-20240327165949732](D:\VScode\gitcode\JieIYu_Markdown_1\问题记录\常见问题及解决方法\AGV梯控使用说明.assets\image-20240327165949732.png)

- 运行楼层
  - 区域 
    - 电梯在要运行的区域
  - 前门设置
    - 入口等待点
      - AGV 进入电梯将在此等待电梯到达以及开门信号。
    - 出口等待点
      - AGV 出电梯到达此点则为该跨楼层电梯任务完成。
    - 交接范围
      - 电梯范围，设备在电梯范围内将无法进行释放。范围选择需包含电梯点、入口等待点以及出口等待点。
    - 是否启用

## 2. AGV 与梯控使用注意事项

###  2.1 人员与 AGV 混用问题

优先人让车，AGV 到达等待点进行呼梯后，会发出保持开门的信号，直到 AGV 进入电梯。

若此时 AGV 被人工干预且位置不变，电梯将保持开门且不动。

电梯卡主第一时间解决办法：

1. 将 AGV 设备推出电梯范围，约推出电梯等待点两个二维码距离（约 3m），如图所示红框范围内为电梯区域。推出该区域后电梯会自动释放（需保持设备在线且已定位）。电梯释放成功会自动关闭电梯门，且电梯内开门按钮释放。

![image-20240327165401832](D:\VScode\gitcode\JieIYu_Markdown_1\问题记录\常见问题及解决方法\AGV梯控使用说明.assets\image-20240327165401832.png)

​	2. 等待电梯口无 AGV 后人员再进行电梯使用（人让车）。

## 3. 电梯调试的说明

### 3.1 电梯调试方式：

1. 电梯转换板输出的是RS485协议 ，查看附件： NC600系列.zip ，转换板连接大华提供的 康海601模块，把RS485协议转换成 tcp协议
2. 参考NC600系列.zip 的使用说明；配置好电脑、ip、波特率：固定波特率：38400bps等信息，准备进行调试
   如果现场电梯直接提供网络协议（IP加端口控制电梯），就直接跳过1，2步骤，直接进行3，使用网络调试测试协议。
   3．使用附件 网络调试助手.Zip 进行tcp协议的发送。(附件的 .exee文件修改为 .exe文件使用)
3. 参考附件 电梯协议对接协议；发送16字节后的协议信息；进行电梯控制。
4. 如可以使用协议中的指令进行开关门操作、召梯（内召、外招），接入平台调度，就没有问题。

此处说明一下跨区域的地图说明：

跨区域地图注意点：

①、跨区域的多副地图，二维码不能相同。不同区域二维码前缀不能相同！

②、多副地图的电梯点二维码值需要相同。电梯轿厢内部的二维码点位，存在于多个区域，点位也需要相同，所有相同的二维码前缀必须是 "00" 开头。

③、电梯点、电梯的出入口点、电梯出入口点到达电梯内部的途径点，都务必是交接点属性。

④、从电梯内的点位出电梯到达任务的目标点，务必经过出口点才能去到目标点，不能存在其他的联通性。
